# 模型先生监控规格

## 数据原则

- 视频列表永久保留，只增加，不因主页下架或视频删除而移除。
- 博主互动事件永久保留，只增加，不修改，不删除。
- 前 20 条评论属于当前快照，允许在监控期内更新；周期结束后保留最后一次成功快照。
- 每个新视频监控 48 小时；删除确认后提前结束并冻结最后快照。

## 新视频流程

1. 每 5 分钟读取主页视频 ID；只有“未知 ID 位于主页最新区域”且“详情发布时间晚于本轮开始前数据库中的最新视频”同时成立，才确认新视频。详情或发布时间暂不可用时留到下轮重试，不使用抓取时间代替发布时间。
2. 发现新视频后立即创建卡片并保存视频基本信息。
3. 排队执行一次视频转录，使用 Whisper `small` 与完整 16kHz 单声道音频。
4. 立即进入评论、互动和统计监控，不等待转录结束。

## 监控频率

- 博主主页（发现新视频）：每 5 分钟。
- 以下频率仅用于已经发现的新视频详情、统计、评论和互动：
- 发布后 0～3 小时：每 3 分钟。
- 发布后 3～24 小时：每 20 分钟。
- 发布后 24～48 小时：每 60 分钟。
- 超过 48 小时：结束周期，冻结最后成功快照。

每次视频轮询读取点赞数、评论数，并读取热度排序前 100 条一级评论用于发现博主点赞、博主一级评论和内嵌作者回复。看板仍只保存和展示前 20 条评论快照及其完整回复，避免公开数据无谓膨胀。前 100 条中一旦发现互动，立即保存评论 ID 和作者互动事件，并进入独立评论 ID 生命周期跟踪；后续不再依赖它是否仍位于前 100 条。

## 分层容错

- 视频详情成功后立即保存统计，不因后续评论或回复接口失败而丢弃本轮统计。
- 前 20 条评论接口暂时失败时保留上次成功快照，下个正常轮询周期继续重试。
- 单条回复线程失败时只保留该线程上次内容，并在下轮继续重试；不得中断其他评论、互动或视频统计。
- 接口空响应、单接口网络异常和普通数据异常不得触发全局暂停。只有明确验证码、登录失效、403 或 429 才触发全局暂停。
- 日志必须标明失败阶段：视频详情、前 20 条评论或具体一级评论回复线程。
- 同一轮第一次遇到完整回复接口的空响应或网络异常后，停止本轮其余完整回复请求，统一保留旧快照并在下轮重试，避免连续请求放大风控；详细日志只记录首次失败摘要。
- 多个活动视频逐条隔离处理：某一视频的普通数据或单视频会话异常只给该视频安排 3 分钟后重试，并继续处理同轮其他到期视频。只有明确验证码、登录失效或全局网络异常才中断后续视频并触发全局暂停。

## 博主互动

互动类型包括：

- 博主点赞别人评论。
- 博主回复别人评论。
- 博主本人发布一级评论。

前 100 条评论负责发现新互动。一级评论接口直接提供作者点赞标记和界面展示用的内嵌回复，因此发现阶段不需要逐一展开 100 条评论；只有发现作者信号后，才按一级评论 ID 请求并保存完整回复线程。发现后保存视频 ID 和一级评论 ID，后续直接通过评论 ID 请求完整回复线程，不再依赖它是否仍在前 100 条中。

互动线程保存原评论和该层全部回复，并在看板中支持展开。取消点赞、删除回复、删除评论或删除视频都不能删除已记录的重要事件。

每条互动线程拥有独立生命周期。后续按视频 ID 与一级评论 ID 直接跟踪整层回复；在网络、登录和验证码均正常的前提下，接口连续 3 次明确返回线程不存在，或原本非空的线程连续成功返回 `total=0`，则只冻结这一条线程并保留最后一次成功快照。其他互动线程继续跟踪。HTTP 200 空响应、超时、限流、验证码和登录异常不累计次数。视频结束 48 小时周期或确认删除时，仍统一冻结该视频剩余的活动互动线程。

## 删除确认

只有网络正常、登录有效且没有验证码时才进行删除判断。活动视频同时满足“主页不存在”和“详情不可用”，连续 3 次后标记删除。验证码、登录失效、限流和网络异常不累计删除次数。

删除后：

- 停止该视频所有监控和转录重试。
- 保留最后一次成功统计、前 20 条评论、转录和全部互动。
- 卡片永久保留，封面区域显示“已删除”。

## GitHub 同步

- 以监控程序启动时间为基准，每 3 小时检查一次。
- 有公开数据变化时提交并推送 `dashboard/public/data/videos.json`。
- 没有变化时跳过提交。
- GitHub 同步周期与视频发布时间、检测时间无关。

## 输出

终端默认每个动作一行，例如：新视频已发现、卡片已生成、视频已转录、视频读取成功、互动已记录、GitHub 已同步、视频已结束 48 小时周期。

终端中的视频标识使用“发布时间 + 视频 ID”，不得使用主页猜测文案；纯数字或点赞数样式的主页文本不能作为视频文案。

接口地址、响应体和调用栈只进入日志文件，不在终端刷屏。
